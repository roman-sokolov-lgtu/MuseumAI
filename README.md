# MuseumAI - Умный Музейный Ассистент

Программный комплекс для музеев на основе искусственного интеллекта (LLM). Позволяет посетителям сканировать QR-коды экспонатов и вести осмысленный диалог с виртуальным гидом.

## 📂 Структура проекта

* `backend/` — Серверная часть (FastAPI, Python). Отвечает за логику ИИ (Gemma 2), API, работу с базой данных и обработку сообщений.
* `admin-panel/` — Веб-панель администратора (React, Vite). Для управления экспонатами, генерации QR-кодов и просмотра аналитики диалогов.
* `mobile-app/` — Мобильное приложение для посетителей (React, Capacitor). Устанавливается на Android. Содержит сканер QR-кодов, чат с ИИ, синтез и распознавание речи.
* `docker-compose.yml` — Конфигурация для быстрого развертывания сервера и базы данных (PostgreSQL).

## 🚀 Как запустить проект

### 1. Серверная часть и База данных (Docker)
Убедитесь, что у вас установлен Docker Desktop.

Перед первым запуском необходимо создать файл конфигурации окружения `.env` из шаблона. В корневой директории проекта выполните:
```bash
# Для Linux / macOS:
cp .env.example .env

# Для Windows (PowerShell):
Copy-Item .env.example .env
```

При необходимости настройте параметры подключения к БД или адрес Ollama в созданном `.env`. Затем в корневой папке проекта выполните:
```bash
docker-compose up -d --build
```
Это поднимет:
- Базу данных PostgreSQL (порт `5432`)
- Бэкенд FastAPI (порт `8000`)
- Панель администратора (порт `3000`)
- Утилиту Adminer для управления БД (порт `8080`)

### 2. Подключение мобильного приложения
Приложение должно быть в одной локальной сети с сервером (например, подключено к Wi-Fi хот-споту ноутбука). Узнайте IP-адрес вашего ноутбука в этой сети (например, `192.168.137.1`).

Перейдите в папку `mobile-app`:
1. Установите зависимости: `npm install`
2. Настройте файл `.env`, указав IP-адрес ноутбука:
   ```env
   VITE_API_URL=http://<IP-АДРЕС-НОУТБУКА>:8000/api
   VITE_STATIC_URL=http://<IP-АДРЕС-НОУТБУКА>:8000
   ```
3. Соберите веб-часть: `npm run build`
4. Синхронизируйте с Capacitor: `npx cap sync android`
5. Соберите APK (требуется Java 21):
   - Откройте папку `mobile-app/android` в Android Studio и нажмите Build APK.
   - ИЛИ через консоль (предварительно указав путь к Java 21): 
     ```bash
     $env:JAVA_HOME="C:\Program Files\Microsoft\jdk-21.0.11.10-hotspot"; .\gradlew.bat assembleDebug
     ```

Готовый файл приложения будет лежать по пути:
`mobile-app\android\app\build\outputs\apk\debug\app-debug.apk`

## ⚙️ Настройка переменных окружения бэкенда (`.env`)

В корне проекта находится файл `.env`, который содержит ключевые параметры конфигурации бэкенда:
* `POSTGRES_USER` / `POSTGRES_PASSWORD` / `POSTGRES_DB` — данные для создания базы данных в Docker.
* `DATABASE_URL` — строка подключения к СУБД PostgreSQL.
* `JWT_SECRET_KEY` / `JWT_ALGORITHM` / `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` — настройки генерации и валидации JWT-токенов администратора.
* `CORS_ALLOWED_ORIGINS` — список доверенных источников для обхода CORS-ограничений (включая локальные хосты и IP-адреса для мобильного приложения).
* `OLLAMA_URL` — адрес API-сервера нейросети Ollama (по умолчанию `http://host.docker.internal:11434`).
* `OLLAMA_MODEL` — название используемой языковой модели (по умолчанию `gemma2:9b`).
